O Ruby on Rails, tambem conhecido como Rails é um Framework livre que promete aumentar velocidade e facilidade no desenvolvimento web, uma vez que é possível criar aplicações com base em estruturas pré-definidas oferecendo uma estrutura robusta e coesa para construir aplicações web eficientes e escaláveis.

O Ruby on Rails é um projeto de código aberto escrito na linguagem de programação Ruby. Criado por David Heinemeier Hansson (DHH), As aplicações criadas utilizando o framework Rails são desenvolvidas com base no padrão de arquitetura MVC (Model-View-Controller).

O "Rails Way" é a filosofia de design que permeia o framework de desenvolvimento web Ruby on Rails e encoraja os desenvolvedores a seguir padrões estabelecidos para construir aplicações web de maneira eficiente e consistente. Quando seguido, o "Caminho Rails" facilita muito na criação de aplicações mas caso ignorado, torna o uso do Rails dificil e complicado. "Para programar em Rails, pense como Rails"

DRY: don't Repeat Yourself é um princípio fundamental de desenvolvimento de software que enfatiza a importância de evitar a duplicação de código. Assim o código terá de ser encapsulado e reaproveitado o desenvolvimento. O deixando mais limpo e fácil de manter

Convention over Configuration: as configurações são baseadas em convenções predefinidas, permitindo que os desenvolvedores se concentrem mais na lógica de negócios da aplicação.

Rails é um "meta-framework", composto por diversos frameworks, com os principais deles sendo:

Active Record: O Active Record é uma camada de mapeamento objeto-relacional (object-relational mapping layer), responsável pela interoperabilidade entre a aplicação e o banco de dados e pela abstração dos dados.

Action Pack: Compreende o Action View (geração de visualização de usuário, como HTML, XML, JavaScript, entre outros) e o Action Controller (controle de fluxo de negócio).

Action Mailer: O Action Mailer é um framework responsável pelo serviço de entrega e até mesmo de recebimento de e-mails. É relativamente pequeno e simples, porém poderoso e capaz de realizar diversas operações apenas com chamadas de entrega de correspondência.

Active Support: Active Support é uma coleção de várias classes úteis e extensões de bibliotecas padrões, que foram considerados úteis para aplicações em Ruby on Rails.

Action WebServices: Provê uma maneira de publicar APIs interoperaveis com o Rails, sem a necessidade de perder tempo dentro de especificações de protocolo. Implementa WSDL e SOAP. O Action Web Service não estará mais presente na versão 2.0 no Rails, visto que o mesmo está voltando-se para a utilização do modelo REST. Mesmo assim, aos ainda interessados em utilizá-lo, será possível fazê-lo através da instalação de um plugin.

Alem de diversas outras Gems que podem ser adicionadas ao projeto e são uma parte integral do ecossistema do Ruby on Rails, onde são frequentemente usadas para estender as capacidades do framework.

Framework: é uma abstração que une códigos comuns entre vários projetos de software provendo uma funcionalidade genérica. Um framework pode atingir uma funcionalidade específica, por configuração, durante a programação de uma aplicação. É o framework quem dita o fluxo de controle da aplicação.

Gems: são pacotes de software que encapsulam e distribuem funcionalidades reutilizáveis. Elas são essenciais para a comunidade Ruby, pois facilitam o compartilhamento e o uso de código entre projetos. As gems podem conter desde pequenas bibliotecas até aplicações completas e são gerenciadas pelo RubyGems, que é o sistema de gerenciamento de pacotes para Ruby.
